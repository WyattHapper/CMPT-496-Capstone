Frequently Asked Questions


        

            
            
        


        


            
Click to learn more about app functionality:


            


                

                    
                        Run Analysis
                    

                    

                        The Run Analysis section allows you to execute different
                        analysis operations on a selected codebase.
                        



                        The Full Codebase Analysis Pipeline runs all available
                        analysis stages automatically. This includes:
                        



                        • Creating a searchable source code database

                        • Generating file-level summaries

                        • Building a summary database

                        • Performing directory-level analysis

                        • Extracting business rules

                        • Validating discovered business rules

                        • Generating unit tests

                        • Creating UML diagrams


                        


                        Individual analysis steps can also be run separately if
                        you only need a specific output. This is useful when
                        updating an existing analysis or troubleshooting a
                        particular stage.
                    

                



                

                    
                        Viewing Insights
                    

                    

                        The View Insights section allows you to explore the
                        information generated during analysis.
                        



                        Available insights may include:
                        



                        • File summaries describing individual source files,
                        classes, functions, and dependencies



                        • Directory summaries explaining the purpose and
                        responsibilities of project folders



                        • Business rule reports containing discovered system
                        logic and requirements



                        • Source files and generated artifacts created during
                        analysis



                        • UML diagrams representing relationships between
                        components
                        



                        Use the sidebar navigation to select the type of insight
                        you would like to view.
                    

                



                

                    
                        Setting / Changing API Key
                    

                    

                        The application requires an API key to communicate with
                        the AI model used during analysis.
                        



                        To set your API key:
                        



                        1. Open the API Key page from the main menu.

                        2. Enter your API key into the input field.

                        3. Submit the key to connect it with the application.


                        


                        If an API key already exists, you can choose to keep the
                        current key or replace it with a new one.
                        



                        Make sure your API key is valid and has access to the
                        required AI services before running an analysis.
                    

                



                

                    
                        Submitting File Path
                    

                    

                        Before running an analysis, you must select the codebase
                        that you want the application to analyze.
                        



                        Use the folder selection button to open your computer's
                        file explorer and choose the root directory of your
                        project.
                        



                        The selected folder should contain the source code,
                        configuration files, and other project files you want
                        analyzed.
                        



                        Selecting the project root folder is recommended because
                        it allows the application to understand the complete
                        structure and relationships within the codebase.
                    

                



                

                    
                        What languages are supported?
                    

                    

                        Language support depends on the parsing tools and analysis
                        agents currently configured in the application.
                        



                        The system works best with commonly structured software
                        projects containing recognizable source files,
                        dependencies, and documentation.
                        



                        If a file type is not supported, it may be skipped during
                        analysis while the remaining supported files continue to
                        be processed.
                    

                



            

        



        


            

                
                    What is CheckPoint?
                

                

                    CheckPoint analyzes software codebases using AI agents
                    to generate summaries, business rules, UML diagrams, and
                    other insights about the project.
                

            


            

                
                    Why use CheckPoint?
                

                

                    Checkpoint is designed to make it easier for software
                    engineers to learn about the components of their 
                    desired codebases to create a faster, more convient workflow 
                    when changes are needed to be implemented. Especially in cases 
                    of sparsely documented legacy code.
                

            



            

                
                    How do I analyze a codebase?
                

                

                    Select your codebase using the folder picker, then run the
                    desired analysis operation. CheckPoint will process the
                    project and display generated results.
                

            



            

                
                    Why does analysis take a long time?
                

                

                    Processing time depends on the size of the codebase, number
                    of files, and AI processing requirements.
                

            


            

                
                    What languages are supported?
                

                

                    In this current iteration we only support C# for the full 
                    pipeline, however more work is being done to integrate and 
                    accomodate different languages
                

            


            

                    
                        How long does analysis take?
                    

                    

                        Analysis time depends on the size and complexity of the
                        selected codebase.
                        



                        Larger projects may require more time because the system
                        must process more files, generate more summaries, and
                        perform additional AI-based reasoning steps.
                        



                        Progress bars are displayed during execution to show both
                        overall pipeline progress and progress within individual
                        tasks.
                    

                



                

                    
                        Does the application modify my code?
                    

                    

                        No. The application analyzes your existing codebase and
                        generates additional output files and reports.
                        



                        Your original source files are not modified during the
                        analysis process.
